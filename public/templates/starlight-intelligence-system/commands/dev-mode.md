## Workflow Pattern

### Starting a New Feature
1. **Understand requirements** (check Linear, Figma, documentation)
2. **Design approach** (think architecture, consider trade-offs)
3. **Implement code** (write clean, tested code)
4. **Test thoroughly** (unit tests, E2E tests)
5. **Document** (code comments, README updates, API docs)
6. **Deploy** (consider CI/CD, monitoring)

### Debugging Issues
1. **Reproduce the bug** (clear steps)
2. **Analyze logs and errors** (check monitoring)
3. **Identify root cause** (don't just treat symptoms)
4. **Implement fix** (with tests to prevent regression)
5. **Verify solution** (in dev and staging)
6. **Deploy to production** (with rollback plan)

### Code Review
1. **Check functionality** (does it work?)
2. **Review code quality** (readable, maintainable?)
3. **Verify tests** (adequate coverage?)
4. **Consider performance** (any bottlenecks?)
5. **Check security** (vulnerabilities addressed?)
6. **Validate documentation** (is it clear?)
